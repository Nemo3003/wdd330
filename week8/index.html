Chapter 8 - CSS3 Transforms and Transitions

Transforms

The transform property is quite nice to use! it allows you to translate, rotate, scale, and many other things!
That is why I love CSS3, we have way more control over what we do on the page.
Translate Property

Translation

These functions allow you to move elements left, right, up, or down. They behave similar to position: relative;
So, you have the translate(x,y), in which X represents the horizontal movement and the vertical movement is managed by the Y
Example: transform: translate(45px, -45px);
Or you can just use "transform: translateX(45px);"
Or transform: translateY(-30px);
Something else to keep in mind is that transforms do not work on inline elements.

Scaling

I honestly never used the scale(x,y) function. So! scale(1) would leave the element the same size, scale(2) makes it double
scale(0.5) would halve them and so on
An example would be: transform: scale(1.5, 0.25)
And of course, you can also use scaleX(x) or scaleY(y)
Something else scaling, as well as translation has no import on the document flow

Rotation

The rotate() function rotates an element around a certain point
So an example could be: transform: rotate(10deg). As you can see, rotate() uses degrees.

Skew

The last transform function is the skew(x,y) function.
An example would be: transform: skew(15deg, 4deg);
Quite useful to modify how we see a text.

Changing the Origin of the Transform

It is possible to control the origin from which our transform is applied. This can be done using the transform-origin property.
An example could be made using: transform-origin: 10% 10%; or transform-origin


Transitions

CSS transitions look smoother than making transitions with JavaScript, especially in mobile devices

  • transition
    • div {
  transition: width 2s, height 4s;
}
  • transition-delay
    •             div {
  transition-delay: 1s;
}
  • transition-duration
    • div{
             transition-duration: 2s;
        }
  • transition-property
    •             div {
  transition-property: width;}
  • transition-timing-function
    •             #div1 {transition-timing-function: linear;}
            #div2 {transition-timing-function: ease;}
            #div3 {transition-timing-function: ease-in;}
            #div4 {transition-timing-function: ease-out;}
            #div5 {transition-timing-function: ease-in-out;}

The transition property is a shorthand property for setting the four transition properties into a single property

The transition-delay property specifies a delay (in seconds) for the transition effect.

The transition-duration property specifies how many seconds or milliseconds a transition effect takes to complete

The transition-property specifies the name of the CSS property the transition effect is for

The transition-timing-function property specifies the speed curve of the transition effect.
This property can have the following values:
ease - specifies a transition effect with a slow start, then fast, then end slowly (this is default)
linear - specifies a transition effect with the same speed from start to end
ease-in - specifies a transition effect with a slow start
ease-out - specifies a transition effect with a slow end
ease-in-out - specifies a transition effect with a slow start and end
cubic-bezier(n,n,n,n) - lets you define your own values in a cubic-bezier function

Animations

Animations unlike transitions allow you to control each step of an animation.
This is preferable when working with mobile.

Keyframes

To control each step of an animation, you need to use keyframes.
This is an example: 

            @keyframes moveRight { 
             from {
                transform: translateX(-50%);
            }
            to {
                transform: translateX(50%);
            }
            }
            
            @keyframes appearDisappear { 
                 0%, 100% {
                opacity: 0;
            }
                20%, 80% {
                opacity: 1;
            }
            }

            @keyframes bgMove {
                100% {
                background-position: 120% 0;
            }
            }

Canvas, SVG, and Drag and Dro

Canvas

With the Canvas API we can draw anything!
Canvas API can also be used for altering videos. Things like green screens and visual effects added to videos.
It is important to note that a new event listener should be added to react to the play event firing on the video element.
Canvas is cool and all but I feel like this is just an API made to spend some time doing something fun with Javascript.
To be honest, I hate to manipulate stuff with canvas.

SVG

SVG stands for "Scalable Vector Graphics". They maintain quality as you make them bigger or smaller.
ViewBox attribute defines the starting location, width, and height.
Circle element defines the a circle, cx, cy, X and Y coords of the
center of the circle.

Räphael Library

Raphaël is an open-source JavaScript library that makes drawing and animating with SVG much easier.

Drag and Drop

The drag and drop API workds just like when you drag and drop files in a dropbox.
However, take into consideration that Android does not support this API.
In order to use it, we add the draggable attribute to HTML elements, and set the value to true
