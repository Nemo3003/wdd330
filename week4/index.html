Chaper 15: Modern JavaScript

So, a module is basically a self-containing piece of code that provides functions and methods. Quite similar to an API to be honest
This way you can have your code in a more organized way. Breaking down the code into smaller pieces
ensuring you will have a highly organized project. You can
store those functions or methods inside different folders as well.
It is considered good design to keep code as loosely coupled as possible


Take into consideration the following:

All code in modules is always in strict mode without the need to use
"use strict" and there is no way to opt out of this.
A modules has its own global scope.
The value of this in the top level of a module is undefined, rather than the global object.
You can't use HTML-style comments in modules.

example here of how to import modules:
first function is created in a file
export const PI = 3.1415926;

then in the file you want to use the above mod. you do this:
import { PI } from './pi.js';

You could even create a library of functions to import into another code

Having more than one default export will result in a syntax error.
So as an alternative we could use named exports.
Also, if you need to import multiple functions
you could use the * symbol: import * as stats from './stats.js';
