Debugging in the browser


You can use the alert popup to see what is going on in the browser.
You can use the console to see what is going on in the browser as well as seeing any error or warning. This includes the console.log()
Debugging tools: Most browsers nowadays include debugging tools, such as Google Chrome, Edge, Safari or Firefox
I love the debugger keyword, it is very useful to see what is your program up to.
You can even create your own error using a constructor: const error = new Error("This is an error");
There are many errors:
  • EvalError
  • RangeError
  • ReferenceError
  • SyntaxError
  • TypeError
  • URIError

To throw errors you can use the throw keyword.
This is an example of the usage of the throw keyword: 
    function throwError() {
        throw new Error("This is an error");
    }
    
        function squareRoot(number) {
            'use strict';
            if (number < 0) {
            throw new RangeError('You can't find the square root of
             negative numbers')
            }
            return Math.sqrt(number);
            };

To handle these errors we have to use try and catch blocks. 
        try {
            squareRoot(-1);
        } catch (error) {
            console.log(error.message);
        }

A finally block is always executed, even if an error is thrown. 
        try {
            squareRoot(-1);
        } catch (error) {
            console.log(error.message);
        } finally {
            console.log("This is always executed");
        }

Tests

Tests are a great way to ensure that your code is working as expected.
Usage of tests can be as follows: 
function itSquareRoots4() {
        return squareRoot(4) === 2;
        }

Test driven development


It's the process of writing tests before any code is written.
After the code passes that test, it is refactored to be more efficient
Its workflow is as follows:
  1. Write a test
  2. Write the code that passes the test
  3. Write a test that fails
  4. Write the code that fails the test
  5. Write a test that passes
  6. Write the code that passes the test
